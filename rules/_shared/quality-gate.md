# 质量门控协议（展示前必须通过，不可跳过）

报告生成后，**必须运行自动化质量检测**，通过后才能展示给用户：

```
Step 1: 运行自动化检测
  python3 "$MADNESS_DIR"/scripts/check_report.py --file /tmp/madness_report_draft.md
  → 输出 JSON：score + 各规则得分明细

Step 2: 判定
  IF score >= 80: 通过 → 进入展示流程
  IF score < 80:
    根据扣分明细，回到阶段 B 补充对应内容
    补充后重新生成报告 → 重新运行检测
    最多重试 2 次
    仍不达标 → 标注「质量告警: score=XX」后继续展示

Step 3: 附质检结果到报告末尾（用户可见），格式：
  [质量自检: XX/100 | 证据引用:XX/17 | 可执行步骤:XX/17 | 最佳实践:XX/17 | 空话检测:XX/17 | 两阶段:XX/17 | 摘要-详情:XX/15]
```

## 人工自检清单（通用项）

- [ ] 每个摩擦点有 ≥1 条用户原话引用？
- [ ] 每条改进有可执行步骤（不是空话）？
- [ ] 是否先展示给用户确认，而非直接写入文件？
- [ ] 对照 [bad-cases.md](../bad-cases.md) 中的自检规则？

**自动化检测 score < 80 且重试 2 次仍不达标 → 标注告警后继续，但不可跳过检测步骤本身。**

**如果任何一条人工自检不满足，回到阶段 B 补充，不要直接输出。**
